Descripción del Modelo de Datos

El modelo relacional de la base de datos para "Arrienda tu Finca" está diseñado para soportar todas las funcionalidades del sistema de arrendamiento de propiedades rurales. La estructura de datos garantiza la integridad referencial y optimiza las consultas más frecuentes del sistema.

Las entidades principales incluyen:

  • Usuarios: Almacena información de arrendatarios y arrendadores con roles diferenciados
  • Propiedades: Contiene detalles de las fincas disponibles para arrendar
  • Solicitudes: Gestiona las peticiones de arriendo entre usuarios
  • Pagos: Registra las transacciones financieras del sistema
  • Calificaciones: Permite el sistema de reseñas bidireccional
  • Ubicaciones: Maneja la información geográfica de las propiedades

Las relaciones entre entidades soportan la lógica de negocio completa, incluyendo restricciones de integridad que garantizan que un usuario no pueda calificar una propiedad sin haber completado un proceso de arriendo exitoso.

Descripción del diagrama

El diagrama de despliegue para "Arrienda tu Finca" muestra la arquitectura física del sistema y cómo los componentes de software se distribuyen en la infraestructura de hardware. La configuración garantiza alta disponibilidad y escalabilidad para soportar múltiples usuarios concurrentes.

Los nodos de despliegue principales incluyen:

  • Servidor Web: Aloja la aplicación frontend React y sirve contenido estático
  • Servidor de Aplicaciones: Ejecuta la API REST backend con balanceador de carga
  • Servidor de Base de Datos: Contiene el motor PostgreSQL con configuración maestro-esclavo
  • Servidor de Caché: Implementa Redis para optimizar consultas frecuentes
  • CDN: Distribuye imágenes y archivos estáticos globalmente
  • Servicios Externos: Integra pasarelas de pago y APIs de geolocalización

La arquitectura implementa contenedorización con Docker y orquestación mediante Kubernetes, permitiendo escalado automático según la demanda. Los protocolos de comunicación incluyen HTTPS para el frontend, REST API sobre SSL para el backend, y conexiones cifradas TCP para la base de datos.

El despliegue considera aspectos de seguridad como firewalls, VPNs para acceso administrativo, y segregación de redes que garantizan que los datos sensibles de pagos y usuarios estén protegidos en capas de red separadas con acceso restringido.
