REQUERIMIENTOS

En esta sección se definen los requerimientos que guían el alcance funcional y de calidad del sistema. Se dividen en dos categorías: Requisitos Funcionales y Requisitos no Funcionales.

A continuación se presentan los requerimientos del proyecto. Los funcionales describen qué debe hacer el sistema, y los no funcionales las restricciones de calidad y comportamiento esperado.

Requisitos Funcionales

Los requisitos funcionales describen las capacidades y comportamientos específicos que el sistema debe ofrecer. Ejemplos ilustrativos:

  • El usuario debe poder registrarse en la plataforma.
  • El sistema debe permitir iniciar sesión con credenciales válidas.
  • El administrador debe poder crear, editar y eliminar recursos.
  • El sistema debe mostrar una lista filtrable de elementos.
  • El usuario debe recibir notificaciones ante eventos importantes.
  • Se debe poder restablecer la contraseña mediante correo electrónico.

Requisitos No Funcionales

Los requisitos no funcionales establecen criterios de calidad, rendimiento, seguridad y usabilidad. Ejemplos ilustrativos:

  • La aplicación debe responder en menos de 2 segundos en condiciones normales.
  • El sistema debe estar disponible el 99.5% del tiempo.
  • Las comunicaciones deben ser cifradas usando HTTPS.
  • La interfaz debe cumplir con estándares de accesibilidad WCAG nivel AA.
  • El sistema debe escalar hasta 500 usuarios concurrentes.
  • Las acciones críticas deben quedar registradas en auditoría.

Notas Adicionales
