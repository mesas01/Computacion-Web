Pruebas del proyecto

En esta sección se buscará validar el correcto funcionamiento de las funcionalidades desarrolladas tanto en el frontend (Angular) como en el backend (Spring REST) de la plataforma, asegurando que cumplan con los requerimientos funcionales definidos en las historias de usuario.

Herramientas y enfoque de pruebas

Las pruebas se realizarán utilizando herramientas como Postman para el backend y Jasmine/Karma para el frontend, asegurando que todas las funcionalidades estén debidamente probadas y documentadas.

Tipos de pruebas

Pruebas de Back-End

  • Pruebas unitarias: Verificarán el correcto funcionamiento de componentes individuales.
  • Pruebas de integración: Verificarán la interacción entre diferentes módulos del sistema.
  • Pruebas de servicios REST: Validan que los endpoints del backend respondan correctamente a las solicitudes HTTP, asegurando que los datos enviados y recibidos sean los esperados y que se manejen adecuadamente los posibles errores.
  • Pruebas de autenticación y autorización: Se enfocan en verificar que solo los usuarios registrados y con los permisos adecuados puedan acceder a ciertas funcionalidades o recursos de la plataforma. Estas pruebas validan el correcto manejo de sesiones, el acceso restringido a información sensible y la protección contra accesos no autorizados.

Pruebas de Front-End

  • Pruebas unitarias: Verificarán el correcto funcionamiento de componentes individuales.
  • Pruebas de componentes: Validarán el buen funcionamiento de los botones, inputs y formularios del sistema.
  • Pruebas de interfaz de usuario: Asegurarán que la interfaz sea intuitiva y cumpla con los estándares de usabilidad.
  • Pruebas de rendimiento: Evaluarán la capacidad del sistema para manejar cargas de trabajo específicas.
  • Pruebas de Servicios HTTP: Se encargan de verificar que las peticiones realizadas desde el frontend hacia el backend se gestionen correctamente. Estas pruebas comprueban que los datos enviados y recibidos sean los esperados, que se manejen adecuadamente los errores y que la comunicación entre cliente y servidor sea segura y eficiente.

Herramientas Utilizadas

Back-End:

  • JUnit 5, Mockito: Estas herramientas se utilizaron para crear y ejecutar pruebas unitarias y de integración en el backend, permitiendo simular comportamientos, validar la lógica de negocio y asegurar que los componentes funcionen correctamente de manera aislada y en conjunto.
  • Postman / Insomnia: Estas herramientas se emplearon para probar manualmente los endpoints de la API, permitiendo enviar solicitudes HTTP, validar respuestas, simular distintos escenarios y documentar los resultados de las pruebas tanto en el desarrollo como en la integración del backend.
  • Spring Boot Test: Esta herramienta permite realizar pruebas de integración y de contexto en aplicaciones Spring Boot, facilitando la simulación de entornos reales, la carga de contextos de aplicación y la validación del comportamiento de los componentes y servicios en conjunto.

Front-End:

  • Jasmine/Karma: Estas herramientas se emplearon para realizar pruebas unitarias y de integración en el frontend desarrollado con Angular, permitiendo automatizar la validación de componentes, servicios y módulos, así como asegurar que la interfaz y la lógica de la aplicación funcionen correctamente ante diferentes escenarios.
  • Protractor: Se empleó para realizar pruebas end-to-end (E2E) en el frontend desarrollado con Angular. Esta herramienta permitió simular el comportamiento real del usuario en la plataforma, interactuando con elementos de la interfaz como formularios, botones y rutas de navegación. Gracias a Protractor, se validó que la aplicación funcionara correctamente desde la perspectiva del usuario final, asegurando la integración entre el frontend y el backend.
  • Postman / Insomnia: Se utilizaron para probar manualmente los endpoints del backend desarrollados con Spring REST. Estas herramientas permitieron verificar que cada servicio respondiera correctamente a las solicitudes HTTP, validar los datos enviados y recibidos, probar distintos escenarios (como errores o peticiones inválidas), y asegurar que el comportamiento del backend cumpliera con los requerimientos funcionales definidos en las historias de usuario.

Despliegue:

  • Docker + Kubernetes de la Universidad: Estas herramientas se utilizaron para el despliegue y la orquestación de los servicios del proyecto, permitiendo simular un entorno de producción real y facilitando la gestión, escalabilidad y monitoreo de la aplicación durante las pruebas y el despliegue final.
